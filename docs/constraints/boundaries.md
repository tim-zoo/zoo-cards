# 边界

这个项目首版的外部输入不多，但仍然要在边界处把数据收紧，而不是让松散数据一路传到页面深处。

## 主要外部输入

### 1. 本地内容文件
包括：
- 卡片 MDX 文件
- 路径配置文件
- 分类、标签、相关推荐等静态元数据

这些内容在进入系统时必须完成：
- 字段解析
- 必填项校验
- slug 合法性校验
- 关联对象存在性校验

### 2. URL 参数 / 路由参数
包括：
- 卡片 slug
- 分类或标签筛选参数
- 路径 slug

要求：
- 进入页面前就做合法性判断
- 对不存在内容返回明确结果，而不是在组件内部兜底猜测

### 3. 构建与部署环境配置
包括：
- GitHub Pages 的 basePath / assetPrefix
- 静态导出配置

要求：
- 配置集中管理
- 不把部署细节散落在页面和组件中

## 边界规则

- 原始 MDX / frontmatter 只能在内容装载层被直接读取
- 页面组件只接收经过整理的结构化数据
- 不把未经校验的标签、分类、关联 slug 直接传到展示层
- 路径配置中的每一步都必须能映射到真实卡片

## 对实现的直接影响

- 需要一层明确的内容 loader / parser
- 需要最小 schema 或类型约束来保护内容质量
- 需要在构建时尽量暴露内容错误，而不是等页面运行时才发现
