# 项目约束

## 硬性限制

- 首版必须适合 GitHub Pages 静态部署
- 首版要同时兼顾手机和 PC 的良好体验，响应式设计从一开始纳入
- 首版不依赖数据库和服务端 API 作为前提
- 首版目标是做出一个好看、能分享、能运行的 MVP，而不是正式商用系统

## 明确不要什么

- 不做重后台产品
- 不做账号体系、登录、评论等复杂功能
- 不做数据库驱动的内容管理作为首版前提
- 不做复杂图谱或 3D 炫技可视化
- 不为了未来扩展过度抽象
- 视觉可以可爱，但不能幼稚

## 对工程实现意味着什么

- 优先采用静态生成和内容文件驱动的架构
- 页面与组件设计优先考虑移动端，再增强 PC 体验
- 功能范围要控制在内容浏览、搜索/筛选、相关推荐和探索路径原型内
- 数据组织要为后续扩展留清晰边界，但不要首版引入过重基础设施
- 设计系统应追求有品牌辨识度的“动物园气质”，同时保持信息清晰和专业感
