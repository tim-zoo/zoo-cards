# 领域说明

这个项目的领域不复杂，但一定要尽早把核心对象讲清楚，否则后面页面、内容文件和探索路径会越写越散。

## 当前建议优先沉淀的领域文档

- `content-model.md` —— 卡片、分类、标签、路径等核心对象定义

## 当前领域范围

首版先聚焦一个窄领域：
- **AI 安全知识卡片**

在这个范围内，产品核心不是“资讯流”，而是“知识单元 + 探索路径”。

## 当前最重要的领域问题

- 一张卡片最小应该包含哪些字段
- 分类、标签和相关推荐分别承担什么角色
- 路径和普通卡片浏览的关系如何定义
- 哪些约束应在内容层就保证，而不是交给 UI 临场处理

后续如果范围扩大，再继续拆更细的领域文档。
