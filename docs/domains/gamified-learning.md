# 游戏化学习功能说明

## 目标

在现有知识卡片与探索路径基础上，为泛用户增加一套轻量、低门槛、即时反馈的学习互动机制，让用户在“学完一小段内容后愿意答一题、得到鼓励、继续下一步”。

这个功能的目的不是把产品做成重游戏，而是提升以下核心指标：

- 启动学习的意愿
- 单次学习完成率
- 连续学习和回访意愿
- 关键知识点的记忆与理解

## 目标用户

面向任何人，不假设用户具备 AI 安全背景。

因此设计原则应偏向：

- 低认知负担
- 反馈直观
- 奖励温和但有感知
- 手机上也能快速完成
- 不制造明显考试压力

## 功能边界

首期聚焦轻游戏化闭环，不引入复杂账号体系、实时对战或重社交机制。

MVP 建议只覆盖：

1. 学完小段内容后的单题互动
2. 即时答题反馈
3. 基础积分 / 经验值
4. 连续学习 streak
5. 徽章或完成进度展示

首期暂不做：

- 大型排行榜
- 用户间 PK
- 复杂道具系统
- 强依赖后端的个性化推荐
- 高运营成本的兑换商城

## 核心体验闭环

### 1. 学习触发

用户通过以下入口进入一次学习：

- 卡片详情页中的“学完来一题”
- 探索路径中的阶段节点
- 首页的“今天学什么”推荐卡

### 2. 微答题

每次完成一个小知识单元后，展示 1 道短题：

- 选择题
- 判断题
- 情景题（后续扩展）

原则：题量极少，答题时间控制在 10 到 30 秒内。

### 3. 即时反馈

答题完成后立即返回：

- 是否正确
- 为什么对 / 错
- 关键知识点提醒
- 是否允许重试一次

反馈语气要温和，避免“考试失败感”。

### 4. 奖励发放

反馈页同步展示轻奖励：

- 获得经验值
- 连击 +1
- 今日进度推进
- 解锁徽章（如满足条件）

### 5. 继续引导

奖励后明确给出下一步 CTA：

- 再学一张卡片
- 继续当前探索路径
- 进入今日下一题

## 奖励机制建议

## 积分 / 经验

建议用经验值而非复杂积分商城，首期更自然。

示例：

- 完成阅读：+5 XP
- 首次答对：+10 XP
- 重试后答对：+5 XP
- 连续学习一天：+10 XP

## 连击 streak

适合做轻量留存机制：

- 当天完成 1 次学习互动，streak +1
- 中断后重置或设置温和补签机制（后续再决定）

## 徽章

徽章更适合泛用户的“收集感”。

首期可做示例：

- 初次答题
- 连续 3 天学习
- 完成第一条探索路径
- AI 安全新朋友

## 进度可视化

建议至少提供：

- 今日完成数
- 当前 streak
- 当前 XP 或 XP 进度条
- 已解锁徽章数

## 内容与数据模型建议

为了支持答题和奖励，内容层至少需要新增以下结构：

### 卡片级字段

- `quiz`: 题目对象
- `xp`: 完成该卡默认奖励值（可选）
- `difficulty`: 与答题反馈节奏联动（可选）

### quiz 对象建议

- `question`
- `type`（single / boolean / scenario）
- `options`
- `answer`
- `explanation_correct`
- `explanation_wrong`
- `retry_hint`

### 用户本地状态

首期仍为纯静态站点，先基于本地存储维护：

- 已答题记录
- 今日学习次数
- streak
- 已解锁徽章
- XP
- 最近一次“今天学什么”推荐结果（可选）

后续如需跨设备同步，再评估账号体系或后端。

## 设计原则

1. 奖励服务学习，不喧宾夺主
2. 反馈具体，不只给“对/错”
3. 交互链路要短，不打断阅读
4. 手机端优先，单手可完成
5. 首期以轻收集和自我进度为主，不强推社交比较

## 已确认决策

1. 首期允许答错后重试一次，但最多只重试 1 次
2. 首期不做等级体系，只展示 XP 与 streak
3. 徽章首期只做展示型收集，不绑定内容解锁
4. 首页“今天学什么”采用人工配置为主，简单规则兜底
5. 题目内容不全部塞进 MDX frontmatter，正文与 quiz 结构分离管理

## 实现策略

建议首期采用：

- 本地状态存储
- 单题即时反馈
- XP + streak + 徽章三件套
- 首页增加“今天学什么”轻入口
- 探索路径页增加阶段完成感
- 卡片正文继续由 MDX 承载，题目与奖励配置放独立结构中

这样实现成本较低，但用户感知会比较明显。
