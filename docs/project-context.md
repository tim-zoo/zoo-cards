# 项目背景

## 项目名称

zoo-cards

## 这个项目是做什么的

在 tim-zoo 下做一个有趣、可分享、适合长期扩展的响应式知识卡片网站。它不是传统博客，也不是厚重文档站，而是一个更轻、更有产品感的知识展示与探索网站。首版直接部署到 GitHub Pages。

## 要解决的核心问题

用卡片方式拆解 AI 安全知识，降低阅读门槛，并在卡片体系之上提供 1 到 2 条“今天学什么”探索路径原型，让用户既能随手浏览，也能按主题逐步学习。

## 谁会使用它

对 AI 安全感兴趣的人；想快速理解某个概念、风险或防御思路的人；想在手机上随手浏览、在 PC 上系统探索的人；园长自己以及 tim-zoo 未来面向的朋友、同事和访客。

## 倾向采用的技术

Next.js 静态导出, MDX 内容驱动, GitHub Pages 部署, 响应式优先设计。

## 时间要求 / 交付压力

先做出一个好看、能分享、能运行的 MVP，没有要求首版正式商用。

## 当前交付倾向

MVP 优先，能力覆盖尽量完整，但实现保持轻量，不做过度设计。
