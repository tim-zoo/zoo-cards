# 计划说明

这里放的是正式计划，不是随手记的草稿。

## 目录约定

- `project-plan.md`：项目宏观计划，用来描述阶段划分、优先级和里程碑
- `active/`：当前正在执行的计划
- `archive/done/`：已经完成的计划
- `archive/abandoned/`：已停止、放弃或失效的计划
- `templates/`：新计划起草模板

## 宏观计划和执行计划的区别

- `project-plan.md` 用来回答“整个项目准备怎么推进”
- `active/*.md` 用来回答“当前这个任务具体怎么做”

不要把这两类计划混在一起。

## 什么时候适合先写执行计划

如果任务满足以下任一条件，通常适合先写执行计划：

- 会改多个文件
- 涉及架构或技术决策
- 需要分阶段推进
- 需要分阶段验证
- 预计不是一次小改就能完成

但这不是初始化阶段的硬编码动作。是否进入计划驱动开发，应由后续 code agent 结合任务复杂度判断，并先和用户确认。

## 命名规则

执行计划文件名统一建议使用：

`NNN-short-action.md`

例如：

- `001-init-code-frame.md`
- `002-build-api-skeleton.md`
- `003-add-basic-ui.md`

文件名用英文，文件内容用中文。

## 工作流

1. 先看 `project-plan.md`，了解整体推进方向
2. 确认需要执行计划后，在 `active/` 新建计划
3. 按计划实施并持续更新状态
4. 完成后移到 `archive/done/`
5. 若中止或废弃，移到 `archive/abandoned/`
6. 新任务新建计划，不覆盖旧计划
