# 004-add-gamified-learning-mvp

## 背景

`zoo-cards` 当前已经具备知识卡片、探索路径和基础浏览体验，但还缺少“学完之后发生什么”的互动闭环。为了提升泛用户的参与感、完成率和回访意愿，需要补上一套轻量游戏化学习机制。

本次执行计划聚焦首期 MVP，不追求完整学习平台，而是优先打通“学习 -> 答题 -> 即时反馈 -> 奖励 -> 继续学习”的最小闭环。

## 目标

完成首期游戏化学习功能的方案落地与前端实现准备，使仓库在结构、内容模型和页面交互层面具备开发条件。

## 范围

本次计划包含：

1. 明确答题与奖励的数据模型
2. 补充首批示例内容结构
3. 设计首页 / 卡片页 / 路径页中的学习互动入口
4. 增加本地进度状态管理方案
5. 实现首版即时反馈与奖励展示
6. 补充验证与回归检查

本次计划不包含：

- 登录 / 账号体系
- 后端存储
- 排行榜或用户对战
- 内容运营后台
- 大规模题库生产工具

## 执行步骤

### Step 1. 文档与模型定稿

- 更新领域文档，固定 quiz、XP、streak、badge 等概念
- 确认首期使用本地存储维护用户进度
- 确认题目内容与卡片内容分离管理
- 固定首期规则：允许一次重试、不做等级、徽章仅展示、今日推荐人工配置优先

完成标准：

- 文档明确字段结构和边界
- 首期关键决策全部落文

### Step 2. 内容结构扩展

- 在内容加载层扩展卡片数据结构，支持 quiz 字段
- 为至少 3 到 5 张示例卡片补齐题目与反馈文案
- 视现有实现决定是继续放在 MDX frontmatter，还是转独立内容文件引用

完成标准：

- 本地可正确读取带 quiz 的卡片内容
- 类型定义和解析逻辑通过验证

### Step 3. 本地学习状态管理

- 新增本地状态模块，管理 XP、streak、已答题、徽章等信息
- 处理无痕或首次访问场景的默认值
- 定义奖励计算规则

完成标准：

- 刷新页面后可保留本地进度
- 状态模块与 UI 解耦，便于后续扩展

### Step 4. 页面交互接入

- 在卡片详情页加入答题入口与答题组件
- 在探索路径页加入阶段完成感和学习状态提示
- 首页补“今天学什么”或今日挑战入口

完成标准：

- 用户能从至少一个真实内容流里完成一次学习闭环
- 手机和 PC 端都能顺畅交互

### Step 5. 即时反馈与奖励表现

- 实现答对 / 答错 / 重试后的反馈面板
- 展示 XP 增量、streak 变化、徽章解锁提示
- 控制文案语气，弱化考试感

完成标准：

- 每次答题后都有明确反馈与下一步 CTA
- 奖励可感知，但不显得花哨冗余

### Step 6. 验证与收尾

- 补类型校验和必要的组件测试或 smoke 验证
- 跑 `scripts/validate.sh`
- 更新 README / docs 中与实际功能不一致的部分

完成标准：

- 本地验证通过
- 文档、代码、功能描述一致

## 风险与注意事项

1. 若把 quiz 直接塞进 MDX frontmatter，类型与编辑体验可能受限
2. 纯本地状态无法跨设备同步，但首期可接受
3. 如果奖励反馈过重，可能破坏当前站点的轻内容气质
4. 若首批示例题写得不够自然，用户会感到“被考试”

## 建议实现顺序

推荐先按以下顺序落地：

1. 文档与类型
2. 示例内容
3. 本地状态模块
4. 卡片页交互
5. 首页 / 路径页接入
6. 奖励与文案打磨

## 当前执行口径

1. 首期做“答错后可重试一次”机制
2. 首期不展示等级概念
3. 今日挑战入口先落首页次屏或首屏导览区，不额外做重运营模块
4. 徽章首期使用静态预设
5. 首批示例卡片优先覆盖 prompt injection、tool abuse、guardrails 等基础主题

## 验证清单

- 卡片内容可带 quiz 正常加载
- 答题后能得到正确反馈
- XP / streak / badge 状态可本地持久化
- 首页能看到今日学习入口
- 探索路径能体现阶段完成状态
- `scripts/validate.sh` 通过
